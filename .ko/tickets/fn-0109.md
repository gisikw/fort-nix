# Rekey script skips sops updatekeys when .sops.yaml unchanged — after adding a new host, the first rekey updates .sops.yaml and re-encrypts files, but if interrupted or partially completed, subsequent runs see .sops.yaml as up-to-date and skip updatekeys entirely. Fix: compare each .sops file's embedded recipients against .sops.yaml and run updatekeys on any mismatches, regardless of whether .sops.yaml changed.
